你的代码与正确的滑动窗口解法有几个关键出入点和逻辑错误，下面逐一分析：

### 1. 变量未定义
- 代码中使用了 `n`（如 `while(r <= n - 1)`），但并没有定义 `n`。应该是 `len`，即字符串长度。

### 2. 修改原字符串为整数索引
- `s[i] = s[i] - 'a'` 将字符替换为对应的整数（0~25），虽然之后用 `s[r]` 作为索引是可行的，但改变了字符串原本的含义，容易引起混淆。建议另建一个整型数组或直接计算。

### 3. 左指针收缩逻辑完全错误
- 你的代码中：
  ```cpp
  while(l < r && sum == 26){
      b[s[l]]--;
      ++l;
      if(b[s[l]]==0){
          sum--;
          minlen = min(minlen, l - r + 1);
          break;
      }
  }
  ```
  - 首先，`b[s[l]]--` 减去的是原来左指针位置的计数，但随后立即 `++l`，此时 `l` 已经指向下一个字符。
  - 然后判断 `b[s[l]]==0`，这是在检查**新左指针位置**的字符计数，而不是被移除的那个字符。这会导致 `sum` 的错误更新。
  - 正确做法：应该先记录被移除的字符，减计数后，检查该字符的计数是否变为0，若是则 `sum--`，并记录当前窗口长度（即 `r - l + 1`，注意此时 `l` 是移动前的值）。
  - 此外，这里只收缩了一次就 `break`，而没有持续收缩到窗口不再满足条件。实际上应该用 `while` 循环一直收缩，直到 `sum < 26`，并在每次收缩时更新最小长度。

### 4. 右指针移动顺序可能越界
- 内层 `while(sum < 26)` 中，先操作再 `++r`，当 `r` 已经到达末尾时，继续 `++r` 会导致越界。应该先检查 `r < len`。

### 5. 最小长度更新位置不当
- 应该在窗口满足条件时（即 `sum == 26`）就记录当前窗口长度，而不是在左移后才记录。你的代码中只在 `sum` 减少时才记录，可能遗漏了某些窗口。

### 正确的滑动窗口框架（供对比）
```cpp
int left = 0, right = 0, sum = 0, minLen = INT_MAX;
vector<int> cnt(26, 0);
while (right < len) {
    // 右指针移动
    int idx = s[right] - 'a';
    if (cnt[idx] == 0) sum++;
    cnt[idx]++;
    right++;
    // 当窗口包含所有字母时，尝试收缩左边界
    while (sum == 26 && left < right) {
        minLen = min(minLen, right - left); // 注意此时窗口为 [left, right-1]
        int leftIdx = s[left] - 'a';
        cnt[leftIdx]--;
        if (cnt[leftIdx] == 0) sum--;
        left++;
    }
}
```

### 总结
你的代码思路接近滑动窗口，但细节实现有误，特别是左指针收缩部分。建议按照标准模板重写，并注意边界条件和变量定义。
