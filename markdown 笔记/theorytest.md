# 理论考试错误点记录
## 1. switch-case语法
```cpp
int x=1, y=0, a=0, b=0;

switch(x)
{
    case 1:
        switch(y)
        {
            case 0: a++; break;
            case 1: b++; break;
        }
    case 2:
        a++; b++; break;
}
```
a,b==?

### 疏忽点

- 上来看到break老兴奋了 你再看看第一行case 1:后面有`break`吗?
- 所以第一层switch全部执行一遍 第二层执行一次`a++`
  
## 2. 短路机制
```cpp
设 x、y、t均为int型变量，则执行语句：x=y=3; t= ++x || ++y; 后，y的值为____。
```
### 答案为什么不是4?

- 1.因为逻辑判断是 **从左至右** 的
- 2.**短路机制** : 对 "**||**" 运算来讲只要从左到右判断一个为true就会返回1.  
 同理对 "**&&**" 只要一个为false就返回0

## 优先级分层（从高到低）

### 1. 访问与调用
- 数组下标 `[]`
- 函数调用 `()`
- 成员选择 `.` 和 `->`

### 2. 单目运算符（右结合）
- `++`、`--`
- `!`、`~`
- `sizeof`
- 类型转换 `(type)`
- 取地址 `&`
- 取值 `*`
- 负号 `-`

### 3. 算术运算
- 乘除模：`*`、`/`、`%`
- 加减：`+`、`-`
- 位移：`<<`、`>>`

### 4. 比较与关系
- 大小比较：`<`、`<=`、`>`、`>=`
- 相等比较：`==`、`!=`

### 5. 位运算
- 按位与 `&`
- 按位异或 `^`
- 按位或 `|`

### 6. 逻辑运算
- 逻辑与 `&&`
- 逻辑或 `||`

### 7. 条件与赋值（右结合）
- 条件（三目）`?:`
- 赋值类：`=`、`+=`、`-=`、`*=`、`/=`、`%=`、`<<=`、`>>=`、`&=`、`^=`、`|=`

### 8. 最低层
- 逗号 `,`

## 容易出错的点
- `.` 的优先级高于 `*` → `*p.f` 实际是 `*(p.f)`，而不是 `(*p).f`
- `[]` 高于 `*` → `int *ap[]` 是“指针数组”，不是“数组指针”
- `()` 高于 `*` → `int *fp()` 是“返回 int 的函数指针”，不是“返回 int* 的函数”
- `==` 和 `!=` 高于位运算
- 算术运算符高于位移
- 逗号运算符优先级最低

# C语言变量命名规则

## 基本规则
- **合法字符**：变量名只能包含字母（a-z、A-Z）、数字（0-9）和下划线（_）。
- **开头限制**：变量名必须以字母或下划线开头，不能以数字开头。
  - ✅ 合法示例：`int age;`  
  - ✅ 合法示例：`int _score;`  
  - ❌ 非法示例：`int 123var;`
- **区分大小写**：变量名区分大小写，例如 `myVar` 和 `myvar` 是两个不同的变量。
- **避免关键字冲突**：变量名不能使用语言的保留关键字，如 `int`、`float`、`if`、`while` 等。

---

## 命名规范
- **见名知意**：变量名应清晰表达其用途。
  - ✅ 推荐：`int totalSum;`  
  - ❌ 不推荐：`int x;`
- **命名风格一致**：
  - 驼峰命名法（camelCase）：`studentAge`
  - 下划线分隔（snake_case）：`student_age`
- **常量命名**：常量通常使用全大写字母，单词间用下划线分隔。

    ```c
    #define MAX_SIZE 100
    const int DAYS_IN_WEEK = 7;
